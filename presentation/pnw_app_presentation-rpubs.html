yawpr will transform your typing

yawpr (yet another word predictor for R) is the ultimate prediction algorithm implemented in the R language. It was developed to allow fast real-time word predictions within a simple user interface.

To demonstrate the algorithm, a web application is presented that allows the user to type words and observe the predictions.

Key features of the algorithm:

  • N-gram Language Model trained from 1, 2, 3 and 4-grams
  • Incremental model training for large corpus
  • Fast prediction of the next word (and up to four more)

How does it work?

yawpr is a machine learning algorithm based on a modified version from the N-gram Language Models and Stupid Backoff paper. The model is trained on 1, 2, 3 and 4-gram's, where each n-gram (wn wn-1 … w2 w1, wi = word) is divided into the root (wn wn-1 … w2) and the last word (w1). Sample probabilities are then computed for all n-grams: p(word|root) = count(word|root) / count(root).

The model is trained on 80% from a corpus of 4 million lines and 100 million words, while the remaining 20% is reserved for testing and validation datasets (10% each).

The training dataset is cleaned (removing obscene words, special characters, URLs, twitter handles and punctuation) but other stop words are not removed (so they can be predicted). This training dataset is then processed by computing frequencies and sample probabilities as explained above.

To predict the next word, a search is initiated at the highest n-gram model depending on the root already seen. The highest probability words following the root matching the n-1 words already seen are selected. In doing so, a new probability is calculated by discounting the word count by a discount_factor (parameter to the algorithm). The algorithm recursively moves to the lower order n-gram (by removing the fist word from the root) and repeating the search. A smoothing parameter is applied for lower level probabilities. As we descend the n-gram levels, the search is optimized by only selecting words that could improve the prediction and have not been predicted yet.

Finally, optimization in memory utilization and speed are implemented by using the data.table R package (with indices), encoding words as integers and hashing n-grams roots.

How well does it predict?

yawpr has an accuracy prediction between 22-28% when tested against a randomly selected dataset not used during training. An accurate prediction is defined as “one of the top five word predicted matches the actual next word”.

Multiple models were created with varying sized datasets from the overall training dataset. These models were then validated against a section of the test dataset (due to computing restrictions). The following table displays the accuracies obtained on each:
